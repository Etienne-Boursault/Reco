# Pipeline de collecte « Reco » — dépendances Python (3.12)
# Installer dans un venv : voir tools/README.md

# Stratégie de versioning :
#  - On épingle un INTERVALLE (>=min, <maj-suivant) plutôt qu'une version
#    exacte. Avantages : on profite des correctifs (patch + mineur) sans se
#    prendre une rupture API d'une version majeure.
#  - Le bas de la fourchette est la version minimale réellement testée
#    localement ; le haut exclut la prochaine majeure (semver).
#  - Pour les libs sans versioning semver propre (yt-dlp, anthropic) on
#    laisse une borne basse seulement — leurs maj contiennent souvent des
#    fixes critiques (anti-blocage YouTube, nouveaux modèles).
#  - En cas de bug de reproductibilité, geler les versions exactes dans
#    un `requirements.lock` séparé (pip freeze) — laisser ce fichier
#    permissif pour le développement.

# Étape 1 — lecture du flux RSS Acast
feedparser>=6.0,<7

# Étape 2 — téléchargement + transcription locale
faster-whisper>=1.0,<2     # transcription Whisper en local (CTranslate2, CPU/GPU)
yt-dlp>=2024.0             # téléchargement audio YouTube (si pas d'audioUrl)
requests>=2.31,<3          # téléchargement des enclosures audio HTTP

# Étape 3 — extraction des recos via LLM
anthropic>=0.40            # SDK API Anthropic (modèle claude-opus-4-7)
python-dotenv>=1.0,<2      # chargement de tools/.env (ANTHROPIC_API_KEY)

# Coordination cross-process (review_server ↔ scripts pipeline)

# Tests — parsing HTML structurel (review_server)

# Cache HTTP SQLite pour `refresh_enrichment.py` (ADR 0023).
