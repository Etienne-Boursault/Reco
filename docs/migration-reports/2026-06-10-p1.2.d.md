## Procédure de rollback (si nécessaire)

Si un bug est découvert post-migration :

```bash
# 1. Stopper review_server
pkill -f review_server

# 2. Supprimer les Items + Mentions migrés
rm -rf src/content/items/un-bon-moment
rm -rf src/content/mentions/un-bon-moment

# 3. (Optionnel) Restaurer recos depuis le backup (recos N'ONT PAS été modifiées par la migration,
# donc cette étape n'est utile que si quelqu'un a touché les recos après l'apply)
cp -r tools/output/backups/pre-migration-2026-06-10/un-bon-moment src/content/recos/

# 4. Vérifier
npm run build
pytest tests/
```

**Note** : `src/content/recos/` est **inchangé** par la migration (lecture seule). Le backup n'est qu'une ceinture-bretelles.

## Doublons capturés (215 items dédupliqués)

215 recos legacy ont fusionné dans des Items canoniques existants. Soit ~7.5% du dataset. Exemples typiques :
- Mêmes films recommandés par plusieurs invités dans des épisodes différents
- Mêmes albums évoqués à plusieurs reprises
- Mêmes livres mentionnés en citation puis en reco par un autre invité

→ La page œuvre canonique (Phase 2) exploitera cette dédup pour montrer "X mentions cross-épisodes".
