## Ce que tu peux faire dans le review server

Démarrage :

```powershell
./tools/.venv/Scripts/python.exe tools/review_server.py --source un-bon-moment
```

→ http://localhost:8000/

Sur chaque carte :

- **Valider / Pas une reco** (existait déjà) — passe `status` à validated/discarded
- **✎ Éditer** (nouveau) — formulaire inline (title, creator, types multi-checkbox, et un `<details>` repliable pour éditer les URLs stockées : `externalIds.*`, `watchProviders[]`)
- **🔄 Ré-enrichir** (nouveau, sur film/série/musique/album/artiste) — relance TMDB/Deezer avec `force=True` ; remplace les URLs stockées si TMDB trouve

Comportement précis :

- Édit `title` → liens calculés par `merchants.ts` (Place des Libraires, JustWatch search…) re-tombent à la volée côté site Astro
- Édit `title` + Ré-enrich → TMDB cherche avec le nouveau titre → URLs stockées (`externalIds.tmdb`, `justwatch`, `watchProviders[]`) sont remplacées

```powershell
# === Démarrage des outils ===
./tools/.venv/Scripts/python.exe tools/review_server.py --source un-bon-moment
npm run dev                       # site Astro dev → http://localhost:4321
npm run build                     # site statique → dist/

# === Pipeline (ordre normal) ===
./tools/.venv/Scripts/python.exe tools/fetch_episodes.py --source un-bon-moment
./tools/.venv/Scripts/python.exe tools/match_youtube.py --source un-bon-moment
./tools/.venv/Scripts/python.exe tools/transcribe.py --source un-bon-moment [--guid GUID]
./tools/.venv/Scripts/python.exe tools/extract_recos.py --source un-bon-moment --provider anthropic
./tools/.venv/Scripts/python.exe tools/extract_recos.py --source un-bon-moment --provider openai
./tools/.venv/Scripts/python.exe tools/enrich_tmdb.py --source un-bon-moment    # films/séries
./tools/.venv/Scripts/python.exe tools/enrich_music.py --source un-bon-moment   # albums/musiques

# === Enrichissement ponctuel (test) ===
./tools/.venv/Scripts/python.exe tools/enrich_tmdb.py --source un-bon-moment --limit 5 [--force]

# === Migration (déjà appliquée, idempotent) ===
./tools/.venv/Scripts/python.exe tools/migrate_types.py --source un-bon-moment

# === Tests ===
./tools/.venv/Scripts/python.exe -m pytest -q
./tools/.venv/Scripts/python.exe -m pytest --cov=tools --cov=src

# === Pipeline orchestré ===
./tools/.venv/Scripts/python.exe tools/run_pipeline.py --source un-bon-moment --steps fetch,transcribe,extract
```

Notes utiles :

- **Tuer un serveur 8000 fantôme** : `netstat -ano | findstr :8000` → repère le PID Windows → `taskkill /PID <pid> /F`
- **Cache Astro stale** après modif schéma : `Remove-Item -Recurse -Force .astro, node_modules\.astro` puis `npm run dev`
- **`.env`** : la clé `TMDB_API_KEY` est dans `tools/.env` (audit sécurité ouvert — voir mémoire `reco-security-audit-2026-05.md`)

## Tests LLM local (autre agent)

Documenté dans [`docs/llm-local.md`](./llm-local.md) (483 lignes). Résumé :

- **Cible** : faire tourner un LLM local (`Qwen3-4B-Q4_K_M.gguf` via `llama.cpp` server) sur la machine `llm.local` pour remplacer Anthropic/OpenAI sur l'extraction de recommandations
- **Endpoint** : `http://llm.local:8080/v1` (compatible OpenAI)
- **Conclusion** : techniquement fonctionnel mais **pas encore assez fiable** pour remplacer Claude Haiku / GPT-4o-mini en production — extrait trop de simples mentions comme des recommandations (taux de faux positifs)
- **Statut** : alternative en exploration, **pas activée** dans le pipeline actuel

Le CR détaille la procédure d'install et les résultats du benchmark. À reprendre quand un modèle plus précis sera testé.
